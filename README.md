# HAIntly HH Profile Service (`profile-service`)

Сервис подключённых HH-аккаунтов и резюме пользователя на FastAPI.

> Статус: архитектурная заготовка. Реализация и контракты будут определены OpenSpec changes.

## Ответственность

- связь пользователя HAIntly с одним или несколькими HH-аккаунтами;
- HH OAuth;
- хранение и обновление access/refresh token;
- шифрование и дешифрование HH-токенов;
- получение и хранение доступных резюме;
- выбранное резюме для поиска вакансий.

## Технологии

FastAPI, PostgreSQL, aiohttp и cryptography.

## Интеграции

- HTTP ← `main-be` для OAuth и управления профилями;
- HTTP ← `vacancy-service` для контролируемого доступа к HH;
- HTTPS ↔ HH API.

## Безопасность и границы

- HH-токены шифруются в покое и маскируются в логах.
- Токены не передаются в `main-fe` и не становятся собственностью вызывающего сервиса.
- Резюме считаются персональными данными.
- Сервис не собирает вакансии и не выполняет AI-оценку.
- Доступ к собственной PostgreSQL БД не предоставляется другим сервисам.

Будущая документация должна добавить OAuth callback flow, token refresh policy, env, миграции, команды запуска и тестов.
